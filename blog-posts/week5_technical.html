T5: Getting schooled on class

09/05/2014

Classes seemed really overwhelming at the beginning of this week (let’s be honest, they were pretty crazy last week too). But really they’re just like building a template so that you can be really lazy. And I’m into that. Instead of having to individually type out mostly-identical code, classes allow you to be dry.

So we’re going to make a class called GroceryList (can you guess what it does?). The first thing that we’ll need to do is define the class as class GroceryList. Note that class is not capitalized.

Let’s add a method called initialize, which will be initiated when a new thing is added to our GroceryList. We will assign it three arguments: name, location, price, because we’ll want to know what the item is called, where in the store it is, and how much it costs each time that we add it to our list. We’re going to add these arguments in the initialize method and define them as instance variables. By making them into instance variables, they can be accessed in other methods in our class without affecting inheritence.

Now let’s add something to our list, because we’re out of food and need to go shopping!

As you can see, we need flour, milk, and ice cream... But isn’t it the worst when you’re at the store and you realize you didn’t bring enough money? Let’s avoid that. What we’re going to do is add a class variable @@total_cost and set it to 0, as that will be the starting total. Then we’ll add a method called total_cost that takes no arguments, and have it add the price of each item we add to our list.
